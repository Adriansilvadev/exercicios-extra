Trabalhando com Tabelas Grandes

Nesta página criamos uma tabela um pouco maior que a da página anterior. nela conseguimos usar os princiapis elementos presente em uma tabela semântica. Um aspecto importante na criação de tabelas semânticas é o uso de escopos nos cabeçalhos. O uso do atributo scope em uma tabela HTML é uma indicação do propósito de uma célula de cabeçalho. Isso ajuda a identificar se a célula do cabeçalho se refere a uma linha, coluna, grupo de linhas ou grupo de colunas. O escopo pode ser aplicado aos elementos th (cabeçalho de tabela) em HTML.

Existem quatro valores possíveis para o atributo scope em uma célula de cabeçalho (th):
col: Indica que o cabeçalho da célula se refere a uma coluna.
row: Indica que o cabeçalho da célula se refere a uma linha.
colgroup: Indica que o cabeçalho da célula se refere a um grupo de colunas.
rowgroup: Indica que o cabeçalho da célula se refere a um grupo de linhas.

Consulte o código fonte desta página para verificar o uso deste atributo e das demais configuraçõees e propriedades usadas na criação desta tabela.

Efeito Zebrado

O efeito zebrado em tabelas é uma técnica de design que alterna o estilo de fundo das linhas em uma tabela para tornar a leitura mais fácil e agradável. Ele é geralmente usado para diferenciar visualmente as linhas da tabela, especialmente em conjuntos de dados extensos, tornando mais simples para os usuários acompanhar as informações em diferentes linhas.

Uma técnicas comumente utilizada é criar pseudo-classes. Neste exemplo usaremos a pseudo-classe nth-child: . com os valores odd e even (:nth-child(odd)) (:nth-child(even)). Verifique o código fonte para notar a aplicação desta pseudo-classe. Para simplificar usaremos a mesma tabela para exemplificar a aplicação deste efeito.

Cabeçalhos Fixos

Em algumas ocasiões podemos ter a necessidade de criar uma tabela que tem um conteudo vertical e horizontals muito alongado. Nestes casos, quando o usuário rola a pagina pode ocorrer de o contúdo do cabeçalho ficar escondido, fazendo o usuário ter que retornar ao incio da tabela para verificar qual é o cabeçalho da informação. Por isso, aplicamos na tabela acima o recurso de cabeçalho fixo. Com a aplicação deste recurso o cabeçalho sempre ficará visivel enquento os dados da tabela tiver visivel na tela.

Para isso, definimos a tabela com um position: relative, seletamos o th, que está dentro do tr, que está dentro do cabeçalho, (thead > tr > th) e definimos o position:sticky; top: 0; e background-color:mesma cor do cabeçalho;. Note que o position sticky deixa o cabeçalho fixo apenas até o momento em que a tabela está visível. Verifique o código fonte para notar estas configurações.
