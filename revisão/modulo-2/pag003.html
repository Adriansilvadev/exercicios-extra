Trabalhando com Fontes.

Conforme já dissemos no inicio desta revisão, de modo geral a maior parte do conteúdo de uma pagina web é composta por informações escritas. E assim como no caso das cores precisamos dar atenção às varias possibilidades que temos para aplicar estilos usando os varios formatos e possibilidades de fontes disponíveis atualmemte. Vamos considerar 2 famílias de fontes as 'com serifa' e as 'sem serifa'.

Fontes serifadas

As fontes serifadas tendem a funcionar bem em contextos onde se deseja criar uma sensação de tradição, elegância, autoridade ou formalidade. Elas são frequentemente utilizadas em impressões como livros, revistas, jornais, convites e materiais acadêmicos. Este parágrafo está usando a fonte serifada 'Times New Roman'.

Fontes não serifadas

Fontes não serifadas tendem a funcionar bem em contextos onde a comunicação precisa ser clara e direta. Elas são frequentemente usadas em materiais como páginas da web, interfaces de aplicativos, cartazes, apresentações e outros elementos de design que requerem uma abordagem moderna e limpa. este parágrafo está usando a fonte não serifada 'Verdana'.

Compatibilidade de fontes

Com tanta variedade de fontes, pode acontecer de alguns dipositivos não serem capazes de reconhcer a fonte que escolhemos usar. Por isso podemos e devemos pensar em soluções visando evitar problemas de compatibilidade. Uma solução simples é definir mais de uma alternativa de fonte usando a propriedade font-family. Por exemplo o Visual studio code sugere a seguinte 'safe combination' (Combinação segura), para uma fonte sem serifa: 'Arial, Helvetica, sans-serif'. Essa combinação permite o navegador escolher a fonte que ele melhor se adapta. Podemos encontrar outras 'safe combinations' neste site. inclusive podemos encontrar safe combinations para fontes 'momoespaçadas' (Monospace).

Para simplificar isso e tornar mais facil tornar o conteúdo escrito adaptável para qualque dispositivo podemos dizer qual á a família de fontes que queremos, dai o navegador decide qual fonte daquela familia usar. Assim podemos simplesmente definir a propriedade font-family das seguintes maneiras:
font-family: sans-serif; para fontes não serifadas ou
font-family: serif; para fontes com serifas ou ainda,
font-family:monospace; para fontes mono espaçadas.

Tamanhos das Fontes

O tamanho da fonte a ser usada em nosso site pode ser definida pela propriedade CSS 'font-size'. O tamanho da fonte (font-size) pode ser definido básicamente de 2 maneiras. Por Medidas Absolutas e Medidas relativas.

Medidas Absolutas:

Centimetro
mm:
Milimetro

pt:
ponto:

Paica

Medidas relativas

em:
Medida relativa ao tamanho atual.
ex:
Medida relativa à altura X.
rem
Medida relativa à fonte principal.
vw
Medida relativa à largura da tela.
vh
Medida relativa à altura da tela.
%
Porcentagem

Peso das Fontes

Outra meaneira de personalizar as fontes é alterando seu peso (font-weight). Existem basicamente 2 maneiras de laterarmos o peso de uma fonte, isso pode ser feito usando pesos literais, são eles: lighter,normal, bold, bolder. Também podemos definir o peso com uma variação numérica, são elas: 100, 200, 300, 400, 500, 600, 700, 800, 900. Porem é importante entender que nem todas as fontes possuem essas variações de peso.

Estilo das Fontes

Ainda outra maneira de aplicar estilos variados de fonte é usando a propriedade font-style: com esta propriedade podemos definir como Italic ou normal.

Texto Sublinhado

Por fim temos a opção de sublinhar o texto. A propriedade text-decoration é a responsável por essa alteração. em todos estes casos é importante lembrar quie nem todas as fontes permitem essas alteraçõe.

Shorthand

Depois de aprender como podemos aplicar estilos às fontes usando todas essas propriedades. podemos lembra que há uma maneira simplificada de aplicar todas essas configurações, isso é possivel graças a uma 'shorthand' (forma abreviada). shorthand para aplicar estas propriedades fica assim:
font-style font-weight font-size 'font-family';
font: italic bolder 1.5em 'Arial', sans-serif;

Um lembrete imprtante sobre as Shothands é que podemos diminuir a quantidade de propriedades da shorthand mas precisamos seguir a sequencia. => estilo => peso => tamanho => familia.
